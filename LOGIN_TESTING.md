# 登录测试说明

## 测试环境搭建

### 1. 安装依赖
```bash
# 在项目根目录执行
npm install
```

### 2. 启动开发服务器
```bash
# 在项目根目录执行
npm run dev
```

### 3. 访问系统
- 开发服务器启动后，会显示访问地址，通常是 `http://localhost:5173`
- 在浏览器中访问该地址

## 登录测试

### 1. 登录页面
- 访问 `http://localhost:5173/login` 进入登录页面
- 页面会显示登录表单，包含用户名和密码输入框

### 2. 登录功能测试
1. **输入用户名**：在用户名输入框中输入任意文本
2. **输入密码**：在密码输入框中输入任意文本
3. **点击登录按钮**：点击"登录"按钮
4. **验证登录结果**：
   - 登录成功后，会显示"登录成功"的提示信息
   - 系统会自动跳转到首页 `http://localhost:5173`

### 3. 登录状态保持
- 登录成功后，刷新页面，系统应该保持登录状态
- 点击"退出登录"按钮，系统会跳转到登录页面

## 常见问题排查

### 1. 开发服务器启动失败
- **问题**：执行 `npm run dev` 后，服务器无法启动
- **解决方案**：
  - 检查端口是否被占用
  - 检查依赖是否正确安装
  - 检查 `package.json` 文件是否正确

### 2. 登录页面无法访问
- **问题**：访问 `http://localhost:5173/login` 后，页面无法显示
- **解决方案**：
  - 检查开发服务器是否正常运行
  - 检查路由配置是否正确
  - 检查 `src/App.tsx` 文件中的路由设置

### 3. 登录后无法跳转到首页
- **问题**：点击登录按钮后，显示登录成功，但无法跳转到首页
- **解决方案**：
  - 检查 `src/pages/Login.tsx` 文件中的跳转逻辑
  - 检查 `src/App.tsx` 文件中的路由保护逻辑
